When I was a kid, I enjoyed playing computer games, and I wanted to program my own. My efforts never resulted in anything more complicated than a text adventure.
Today, the Open Source community is filled with game engines and libraries that make it pretty easy to code games. Ironically, by the time these started coming out, I lost most of my interest in computer games. Now, I'm a lot more interested in German-style board games than any video game. I've never owned a first-person shooter in my life. I can no longer get into RPGs like Final Fantasy. And don't get me started on banal online games such as League of Legends, whose only effect is to waste time and deprive people of sleep. The few games I play are strategy games. The Civilization series is the only series from childhood I still enjoy.
So, it's gotten to the point where I usually enjoy making games more than actually playing them. However, they offer me the chance to play around with different ideas, such as unusual gameplay. I've made old-school RPGs that focus on conversation and solving mysteries rather than fighting. I like approaching traditionally game tropes from different perspectives. And I include a lot of dialogue (even in my idle game, not a genre given to much words).

Name Link Engine Description
The Townfolk Cartel Link Phaser.io I never wrote an idle game before, so I made this one as a lark - I wanted to create a game about traditional RPG tropes, but played from a different perspective. It proved more popular than the preceding RPGs (and was a lot easier to make). I actually had to make an open-source contribution to the Phaser engine to get this to work how I wanted. Both the game and the plugin I wrote were discussed in this issue of Phaser World.
Apprentice Wars Link Enchant.js This RPG game used a vastly improved game engine, most notably a world map that used the A* algorithm to move the character to wherever on the screen the player clicked and pointed. I also added standard RPG fare like NPCs on the map and random encounter battles. The story here was more involved, but was not, in my opinion, as focused and straightforward as the storyline in the first one.
The Stranger in Town Link Enchant.js My first traditional RPG game, with some interesting modifications. First, I added a mystery solving engine - the goal of this game is to collect clues, logically combine them to come up with other clues, and solve a problem. Second, I added a conversation mechanism to allow more interesting NPC interaction than traditional RPGs. However, the core gameplay had serious limitations, which were addressed in the second iteration of the game engine. Also, while it had a focused storyline and goal, the dialogue was too wordy.
